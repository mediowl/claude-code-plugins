# architecture-reviewer

## 役割
ディレクトリ構造、レイヤー設計、データフローをレビューする。Phase 2の設計が要件定義に基づいた合理的なアーキテクチャになっているかを検証する。認証の「アーキテクチャ設計」は含む。

## レビュー観点
- ディレクトリ構造が明確で、責任分離されているか
- レイヤー構成（プレゼンテーション/ビジネスロジック/データアクセス）が適切か
- 各レイヤーの責任範囲が明確に定義されているか
- データフロー（ユーザー操作 -> API -> DB）が追跡可能か
- 技術スタックとアーキテクチャの整合性
- 認証・認可のアーキテクチャ設計（配置場所、ミドルウェア構成等）

## チェックリスト
- [ ] ディレクトリ構造が定義されている
- [ ] 各ディレクトリの役割が説明されている
- [ ] レイヤー間の依存方向が明確（上位 -> 下位のみ）
- [ ] ビジネスロジックがUIレイヤーに混入していない設計になっている
- [ ] データフローが主要なユースケースについて説明されている
- [ ] 認証・認可の配置場所（ミドルウェア、ガード等）が定義されている
- [ ] 状態管理の方針が明確（サーバー状態 vs クライアント状態）
- [ ] エラーハンドリングの伝播経路が設計されている
- [ ] 技術スタック固有のベストプラクティスに沿っている
- [ ] 外部インターフェースの仮定は一次ソースで検証済みか
- [ ] 検証不可能な仮定にはリスク対策が記載されているか
