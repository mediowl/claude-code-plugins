# feasibility-reviewer

## 役割
技術スタックとMVP期間の整合性をレビューする。Phase 0で想定された技術構成と開発体制で、提案されたMVPが期間内に実現可能かを検証する。

## レビュー観点
- 技術スタックの選定がプロダクトの要件に適しているか
- MVP期間と機能ボリュームの整合性
- 開発体制（個人/チーム/Claude Code活用）を考慮した工数見積もりの妥当性
- 技術的ブロッカーの有無（未成熟な技術、ライセンス問題、外部依存など）
- 技術スタックの学習コスト（チームが未経験の場合）
- 類似プロジェクトの失敗事例への考慮（同種のプロダクトでよくある失敗パターンが検討されているか）
- 課題と解決策のフィット（定義された課題に対して、提案されたプロダクトが適切な解決策になっているか）

## チェックリスト
- [ ] 技術スタックが明示されている
- [ ] MVP期間が具体的に設定されている（ユーザー指定の期間）
- [ ] 技術スタックとプロダクト要件の適合性が説明されている
- [ ] 明らかな技術的ブロッカーが洗い出されている
- [ ] 開発体制に対してMVP期間が現実的である
- [ ] 外部サービス依存（API、BaaS等）のリスクが考慮されている
- [ ] 技術選定の理由が「慣れている」以外にも記述されている
- [ ] 類似プロジェクトの失敗事例が検討されている（または該当なしの理由が記載されている）
- [ ] 定義された課題と提案されたプロダクトの解決策がフィットしている
