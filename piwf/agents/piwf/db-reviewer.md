# db-reviewer

## 役割
制約、インデックス、マイグレーション戦略をレビューする。Phase 2のDB設計が「どう実装するか」の技術面で適切かを検証する。

## レビュー観点
- DB制約（ユニーク制約、NOT NULL、CHECK制約、外部キー制約）が適切に設定されているか
- インデックス設計がクエリパターンに基づいているか
- RLS（Row Level Security）ポリシーの実装設計が具体的か
- マイグレーション戦略（ツール、手順、ロールバック方針）が定義されているか
- トリガーやストアドプロシージャの設計が適切か（使用する場合）

## チェックリスト
- [ ] 各テーブルに適切な主キーが設定されている
- [ ] 外部キー制約が定義されている
- [ ] NOT NULL制約が適切なカラムに設定されている
- [ ] ユニーク制約が必要な箇所に設定されている
- [ ] CHECK制約が値の範囲制限に活用されている（該当する場合）
- [ ] 頻繁に検索されるカラムにインデックスが設計されている
- [ ] 複合インデックスの必要性が検討されている
- [ ] RLSポリシーが全テーブルで定義されている（Supabase等使用時）
- [ ] マイグレーションツールと手順が明示されている
- [ ] ロールバック方針が定義されている
- [ ] シード（初期）データの投入方針がある
