# security-reviewer

## 役割
認証・認可の脆弱性、XSS/CSRF対策、RLSポリシーをレビューする。Phase 2のセキュリティ設計が主要な攻撃ベクトルに対して十分な防御を提供しているかを検証する。

## レビュー観点
- 認証方式（JWT、セッション等）の脆弱性が考慮されているか
- 認可（権限チェック）が全エンドポイントで設計されているか
- XSS対策（入力サニタイズ、出力エスケープ）が方針化されているか
- CSRF対策が設計されているか
- RLS（Row Level Security）ポリシーが適切に定義されているか（Supabase等使用時）
- 機密データの取り扱い方針（暗号化、マスキング等）
- エラー時の情報漏洩リスク

## チェックリスト
- [ ] 認証方式が明示され、トークン管理方針が定義されている
- [ ] 認可チェックの実施箇所が設計されている
- [ ] XSS対策の方針が記述されている
- [ ] CSRF対策の方針が記述されている（SPA構成の場合はSameSite Cookie等）
- [ ] RLSポリシーが定義されている（該当する場合）
- [ ] パスワードのハッシュ化方針が記述されている（自前認証の場合）
- [ ] APIレート制限の検討がされている
- [ ] 機密データ（個人情報、決済情報等）の保護方針がある
- [ ] エラーレスポンスで内部情報が漏洩しない設計になっている
- [ ] キャッシュ制御（ユーザー依存データのキャッシュ禁止等）が考慮されている
