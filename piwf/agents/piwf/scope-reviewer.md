# scope-reviewer

## 役割
Must機能がMVP期間に収まるかをレビューする。Phase 1の要件定義書において、スコープと開発リソースのバランスが現実的かを検証する。

## レビュー観点
- Must機能の総量がMVP期間内に実装可能か
- 各Must機能の実装難易度・工数が考慮されているか
- 技術スタックの学習コストが工数に含まれているか
- 外部サービス連携（認証、決済等）の統合工数が考慮されているか
- テスト・デバッグ・デプロイの時間が確保されているか
- スコープ超過時の削減候補が明確か

## チェックリスト
- [ ] Must機能の一覧とMVP期間が明示されている
- [ ] 各Must機能に概算の実装難易度・工数が見積もられている（明示的でなくても妥当性を判断可能）
- [ ] Must機能の合計工数がMVP期間の80%以内に収まっている（バッファ考慮）
- [ ] 外部サービス連携の工数が考慮されている
- [ ] 「1人日で終わらない」規模のMust機能が分割可能か検討されている
- [ ] スコープ超過時の対応方針（機能削減 or 期間延長）が記載されている
- [ ] Must機能間の依存関係が把握されている
