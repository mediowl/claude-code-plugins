# datamodel-reviewer

## 役割
エンティティ・リレーション設計の妥当性をレビューする。Phase 1のデータモデル設計が要件を満たしているかを検証する。「何のデータを持つか」の設計レビューに焦点を当て、制約・インデックス等の「どう実装するか」はdb-reviewer（Phase 2）の担当とする。

## レビュー観点
- エンティティが要件定義の機能をカバーしているか
- リレーション（1:N、N:N等）の方向と多重度が正しいか
- 正規化レベルが適切か（過剰な正規化で複雑化していないか、非正規化が必要な箇所がないか）
- 命名規則が一貫しているか
- 将来の拡張性が考慮されているか（ただし過度な汎用化は不要）
- 必要なエンティティの漏れがないか

## チェックリスト
- [ ] 全Must機能に対応するエンティティが定義されている
- [ ] 各エンティティの主要カラムが列挙されている
- [ ] リレーション（1:1、1:N、N:N）が明示されている
- [ ] N:Nリレーションに中間テーブルが定義されている
- [ ] 命名規則（snake_case等）が一貫している
- [ ] ユーザー関連エンティティに適切な識別子がある
- [ ] 論理削除 vs 物理削除の方針が決まっている
- [ ] タイムスタンプ（created_at、updated_at）の方針が定まっている
