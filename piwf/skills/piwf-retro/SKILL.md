# /piwf-retro -- PIWF 振り返りスキル

## Hooks 登録チェック（スキル実行開始時に必ず実施）

このプラグインは以下の hooks を提供しているが、Claude Code はプラグインの hooks を自動登録しない。
スキル実行開始時に、以下の手順で hooks が登録済みか確認せよ。

### チェック手順

1. 以下のファイルを順に確認する:
   - `.claude/settings.json`（プロジェクトスコープ）
   - `~/.claude/settings.json`（ユーザースコープ）
2. いずれかのファイルに以下の **両方** が登録されていれば OK:
   - `hooks.SessionStart` に `check-update.sh` を含むエントリ
   - `hooks.PreToolUse` に `guard-dangerous-commands.sh` を含むエントリ
3. 片方でも未登録の場合、以下の案内を表示してからワークフローを続行せよ:

```
[注意] piwf プラグインの hooks が settings.json に登録されていません。
以下の機能が無効になっています:

- SessionStart hook: プラグインの自動更新チェック
- PreToolUse hook: 危険コマンド（rm -rf, git push --force 等）のブロック

セットアップ方法:
プロジェクトの .claude/settings.json（またはユーザーの ~/.claude/settings.json）に以下を追記してください:

{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/plugin-sources/mediowl-plugins/piwf/hooks/check-update.sh piwf mediowl-plugins mediowl/claude-code-plugins",
            "timeout": 30
          }
        ]
      }
    ],
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "~/.claude/plugin-sources/mediowl-plugins/piwf/hooks/guard-dangerous-commands.sh",
            "timeout": 10
          }
        ]
      }
    ]
  }
}

詳細は piwf の README.md「Hooks のセットアップ」を参照してください。
```

### 注意事項

- hooks が未登録でもワークフロー自体は実行可能。案内表示後にブロックせず続行すること
- 既に登録済みの場合は何も表示せずそのまま進めること

## このスキルの使い方

ユーザーが「振り返り」「retro」「piwf-retro」に言及したら、このスキルに従って行動せよ。

## 前提条件

- PIWFで開発したプロジェクトの実装が完了（またはある程度進行）していること
- `piwf-output/{プロジェクト名}/` に成果物が存在すること

## 実行手順

### Step 1: プロジェクト情報の確認

以下をユーザーに確認せよ:

- プロジェクト名（piwf-output配下のディレクトリ名）
- 実装に使ったワークフロー（dbz-workflow等）
- 実装中に気づいた問題点

### Step 2: ゲート検出問題の一覧

各Phaseのゲートレビュー結果を振り返り、検出された問題を一覧化せよ。

| Phase | 検出した問題 | 対応結果 | 指摘者 |
|---|---|---|---|
| ... | ... | ... | ... |

### Step 3: 定量評価

以下の指標を記録せよ:

- 申し送りリストの消化率: N/N件
- フェーズ内イテレーション回数: Phase Nで N回
- 実装中発見事項: N件

### Step 4: ナレッジ蓄積（次プロジェクト向け）

以下を記録し、次プロジェクトで再利用できる形にせよ:

**ライブラリ対策の更新:**
- 今回ハマったライブラリの問題と回避策
- 次プロジェクトのPhase 3「ライブラリ対策ドキュメント」に反映

**「避けるべき落とし穴」の更新:**
- 今回踏んだ地雷の一覧
- 次プロジェクトのCLAUDE.md「避けるべき落とし穴」セクションに反映

**設計判断の記録:**
- 「なぜその設計を選んだか」「今ならどうするか」
- 次プロジェクトのPhase 1-2での参考資料

### Step 5: プロセス改善提案

PIWFフレームワーク自体の改善提案があれば記録せよ。

## 成果物の出力

ファイル名: `piwf-output/{プロジェクト名}/piwf-retrospective.md`

以下のテンプレートに従って出力せよ。

## 完了条件

- [ ] ゲート検出問題が一覧化されている
- [ ] 定量評価が記録されている
- [ ] ナレッジ蓄積（ライブラリ対策、落とし穴、設計判断）が記録されている
- [ ] 振り返りレポートファイルが `piwf-output/{プロジェクト名}/` に出力されている

## 完了メッセージ

```
振り返りが完了しました。

piwf-retrospective.md は次のPIWFプロジェクトのPhase 0で受け取ることで、
過去の教訓を活かした開発が可能になります。
```
