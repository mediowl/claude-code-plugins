# /piwf-retro -- PIWF 振り返りスキル

## このスキルの使い方

ユーザーが「振り返り」「retro」「piwf-retro」に言及したら、このスキルに従って行動せよ。

## 前提条件

- PIWFで開発したプロジェクトの実装が完了（またはある程度進行）していること
- `piwf-output/{プロジェクト名}/` に成果物が存在すること

## 実行手順

### Step 1: プロジェクト情報の確認

以下をユーザーに確認せよ:

- プロジェクト名（piwf-output配下のディレクトリ名）
- 実装に使ったワークフロー（dbz-workflow等）
- 実装中に気づいた問題点

### Step 2: ゲート検出問題の一覧

各Phaseのゲートレビュー結果を振り返り、検出された問題を一覧化せよ。

| Phase | 検出した問題 | 対応結果 | 指摘者 |
|---|---|---|---|
| ... | ... | ... | ... |

### Step 3: 定量評価

以下の指標を記録せよ:

- 申し送りリストの消化率: N/N件
- フェーズ内イテレーション回数: Phase Nで N回
- 実装中発見事項: N件

### Step 4: ナレッジ蓄積（次プロジェクト向け）

以下を記録し、次プロジェクトで再利用できる形にせよ:

**ライブラリ対策の更新:**
- 今回ハマったライブラリの問題と回避策
- 次プロジェクトのPhase 3「ライブラリ対策ドキュメント」に反映

**「避けるべき落とし穴」の更新:**
- 今回踏んだ地雷の一覧
- 次プロジェクトのCLAUDE.md「避けるべき落とし穴」セクションに反映

**設計判断の記録:**
- 「なぜその設計を選んだか」「今ならどうするか」
- 次プロジェクトのPhase 1-2での参考資料

### Step 5: プロセス改善提案

PIWFフレームワーク自体の改善提案があれば記録せよ。

## 成果物の出力

ファイル名: `piwf-output/{プロジェクト名}/piwf-retrospective.md`

以下のテンプレートに従って出力せよ。

## 完了条件

- [ ] ゲート検出問題が一覧化されている
- [ ] 定量評価が記録されている
- [ ] ナレッジ蓄積（ライブラリ対策、落とし穴、設計判断）が記録されている
- [ ] 振り返りレポートファイルが `piwf-output/{プロジェクト名}/` に出力されている

## 完了メッセージ

```
振り返りが完了しました。

piwf-retrospective.md は次のPIWFプロジェクトのPhase 0で受け取ることで、
過去の教訓を活かした開発が可能になります。
```
